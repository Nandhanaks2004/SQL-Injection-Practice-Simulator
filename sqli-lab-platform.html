FOUNDATION MODULE

SQL Injection — Core Concepts

Before exploiting, you must understand. This module teaches you how SQL works, why injections happen, and how attackers think.

🗄

What is SQL?

Databases store data in tables. SQL (Structured Query Language) is used to read and write that data.

EXAMPLE QUERY
SELECT * FROM users
WHERE username = 'admin';

This query fetches the user whose username is admin from the users table.

⚠️

The Vulnerability

When user input is directly inserted into a SQL query without validation, the attacker can inject SQL code and change the query's logic.

User Input (Malicious)
→
Injected SQL Query
→
Unintended Database Response
🧪

How Injection Changes a Query

✅ NORMAL INPUT
SELECT * FROM products
WHERE category = 'Gifts'
  AND released = 1;

Shows only released Gifts products. Normal behaviour.

⟹
💀 INJECTED INPUT
SELECT * FROM products
WHERE category = 'Gifts'
  OR 1=1--' AND released = 1;

' closes the string · OR 1=1 is always TRUE · -- kills the rest

🔑

The SQL Comment Trick

In SQL, -- marks the beginning of a comment. Everything after it is ignored by the database engine.

WHERE username = 'admin'--
AND password = 'anything';

The password check never runs. Authentication is bypassed.

📚

Types of SQL Injection

Error-based Extracts data via database error messages
UNION-based Appends extra SELECT to retrieve other table data
Boolean-based Asks TRUE/FALSE questions to extract data bit by bit
Auth Bypass Manipulates login queries to skip password checks

LAB 3 · UNION ATTACK

Column Discovery via UNION SELECT

To extract data from other tables using UNION, you must first know the exact number of columns and which columns accept text. This lab teaches you how.

STEP 1
Confirm SQL Injection Exists

First, test if the parameter is injectable by sending a single quote:

TEST PAYLOAD
/filter?category=Pets'
SERVER RESPONSE
500 Internal Server Error — You've broken the SQL syntax. Injection confirmed.
STEP 2
Determine Column Count

A UNION query must return the exact same number of columns as the original query. We test incrementally with NULLs:

' UNION SELECT NULL --
❌ 500 Error (1 column — wrong)
' UNION SELECT NULL, NULL --
❌ 500 Error (2 columns — wrong)
' UNION SELECT NULL, NULL, NULL --
✅ 200 OK — 3 columns confirmed!
Why NULL? NULL is compatible with any data type, so it won't cause type mismatch errors while we're just counting columns.
STEP 3
Identify Text-Compatible Column

Now swap each NULL for a string to find which columns can display text data:

' UNION SELECT 'test', NULL, NULL --
❌ 500 Error — Column 1 is numeric
' UNION SELECT NULL, 'test', NULL --
✅ String 'test' appears on page!
Column 2 is our text column. We can now inject string data (like usernames and passwords) through column 2 and have it displayed on the page.

DEFENSE MODULE

Preventing SQL Injection

Now that you understand how attacks work, learn how to stop them. These are industry-standard defenses used by every secure application.

🛡

1. Parameterized Queries (Prepared Statements)

The most effective defense. User input is never concatenated into the SQL string — it's passed as a separate parameter.

❌ VULNERABLE
query = "SELECT * FROM users
WHERE user = '" + input + "'"
✅ SAFE
query = "SELECT * FROM users
WHERE user = ?"
stmt.setString(1, input)
📦

2. Use an ORM (Object Relational Mapper)

ORMs like Sequelize, SQLAlchemy, or Hibernate generate parameterized queries automatically. You never write raw SQL.

DJANGO ORM EXAMPLE (SAFE)
User.objects.get(username=username)
✅

3. Input Validation & Allow-Lists

Validate and sanitize all user input. Use allow-lists (whitelist only known-good values) rather than blocklists.

StripRemove special characters like ' -- ;
WhitelistOnly allow expected values (e.g., category must be Gifts, Pets, etc.)
RejectReturn error if input doesn't match expected pattern
🔒

4. Principle of Least Privilege

The database account your app uses should have only the permissions it needs. A read-only app should use a read-only DB user.

🚫 No DROP TABLE access for web users
🚫 No access to system tables
✅ Separate read/write accounts
🚨

5. Web Application Firewall (WAF)

A WAF inspects HTTP traffic and blocks requests that match known attack patterns like UNION SELECT or OR 1=1.

Not a silver bullet. WAFs can be bypassed by skilled attackers. They're a layer of defense, not a replacement for secure code.
📊

6. Monitor & Log

Log all database queries and monitor for suspicious patterns. Anomaly detection can catch active attacks in progress.
