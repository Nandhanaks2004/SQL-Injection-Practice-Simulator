# SQL Injection Interactive Learning Lab

## Overview

This project is a browser-based interactive lab designed to help learners understand SQL Injection (SQLi) through practical, visual simulations.

Instead of relying only on theory, this lab demonstrates how SQL queries work, how user input affects them, and how attackers manipulate queries to bypass authentication or extract data.

The focus is on building a clear conceptual understanding of SQL Injection using safe and controlled simulations.

## Objectives

* Explain how databases and SQL queries work
* Demonstrate how web applications interact with databases
* Show how SQL Injection occurs in real scenarios
* Visualize how user input modifies SQL queries
* Provide hands-on interaction to simulate attacks
* Teach secure coding practices to prevent SQL Injection

## Features

* Interactive query simulation with real-time updates
* Login bypass demonstration using SQL Injection
* Visualization of query transformation (before and after injection)
* Highlighted injected payloads for clarity
* Simulations of common SQL Injection techniques
* Simple and structured learning flow from basics to attacks

## Covered Concepts

* Database fundamentals
* SQL basics (SELECT, INSERT, UPDATE, DELETE)
* Query structure and WHERE clause
* How user input is processed in queries
* SQL Injection logic and exploitation
* Common SQL Injection attack types
* Prevention techniques using secure coding practices

## How It Works

The lab is divided into structured sections:

1. Basics
   Introduction to databases and SQL queries

2. Query Flow
   How web applications generate SQL queries from user input

3. Injection Simulation
   Demonstrates how malicious input alters query logic

4. Attack Scenarios
   Includes login bypass and data extraction examples

5. Prevention
   Shows how secure query handling prevents SQL Injection

## Usage

This is a static web application and does not require installation.

To run the project:

* Download or clone the repository
* Open the main HTML file (e.g., `index.html`) in any modern web browser

No additional setup or dependencies are required.

## Important Note

This project is created for educational purposes only.

All vulnerabilities are simulated in a controlled environment using mock data. The application does not perform real attacks and should not be used against real systems.
